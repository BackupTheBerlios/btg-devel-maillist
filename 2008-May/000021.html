Michael Wojciechowski wrote:
> What about hw where you have to emulate floating point in software?
>
> I just wanted you to think about a way of avoiding using floats, if
> possible. If not, then leave it as it is
I see your point, but libcurl itself uses doubles and we depend on it.
Generally we can disable all this part of functionality when building
for the embedded.

Btw, FP emulation code works only when there is an ALU operation to do.
On the daemon side there is only conversion from double to float,
which can be omitted by introducing t_double our type, which consumes
twice more memory.
If we use integer, there will be conversion from double to integer,
which a bit more complicated than between floats.

XML converts floats rather fast (just serializing it's internal
representation, I think,
since internal representation of float-like types is standardized by
IEEE and should be equal on all systems).
